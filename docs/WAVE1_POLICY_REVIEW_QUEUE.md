# Wave 1 policy review queue

## Purpose

第1波9都道府県の現行計画入口は確認済みですが、公式資料を見つけた状態と、政策体系・KPIを一次資料と照合したReviewed状態は異なります。

このキューは、次の作業対象、必要な品質ゲート、作業順の理由を記録し、件数拡大による誤昇格を防ぐための運用台帳です。

機械可読な正本は [`data/catalog/wave1_policy_review_queue.json`](../data/catalog/wave1_policy_review_queue.json) です。

## Reviewed promotion gate

都道府県を`reviewed_data`へ昇格するには、最低でも次を満たします。

1. 現行計画と後継・改定状況を確認済み
2. 政策階層を原文・公式順序で登録
3. KPI名、基準値、目標値、単位、期間、条件を一次資料と照合
4. 欠損、条件型目標、上下限、累計・単年度の違いを保持
5. 再掲指標を重複登録せず、参照として管理
6. 全公開レコードにEvidence Packetを付与
7. source ID、ページ、位置、レビュー判断を保存
8. JSON Schema、参照整合性、重複検査を通過
9. Web lint、TypeScript、本番ビルド、静的出力を通過
10. 公開後Smokeで主要文言・件数・品質境界を確認

## Source status boundaries

- `indexed / verified`：公式資料の存在、URL、現行性を確認済み。本文データは未Reviewed。
- `reviewed`：本文、数値、期間、単位を人が一次資料と照合済み。
- `actuals_linked`：目標と年度実績、公式説明、定義変更を接続済み。
- `published`：公開ゲートと本番確認を通過済み。

## Active work: Hokkaido

北海道は、公式入口に108指標の個票がまとまっているため、福岡県以外で最初のReviewed抽出対象とします。

最初の成果物は次の順で作成します。

1. 計画全体の政策階層カタログ
2. 指標個票の資料・ページ索引
3. 指標名、基準値、目標値、単位、期間の抽出
4. 条件型・累計・単年度・欠損の分類
5. Evidence Packet
6. 自動テスト
7. 北海道公開ページ

資料本文を確認できない項目は推測で補完せず、`needs_review`または未登録のまま残します。
