## Batch quality gates

各バッチは、次の品質ゲートを順に通過します。

1. 現行計画と旧計画・後継計画の境界確認
2. 数値目標表・KPI資料の位置索引
3. 指標本文、値、単位、期間、母集団のReviewed化
4. 一つの指標につき一つ以上のEvidence Packet
5. 年度実績・評価資料との定義照合
6. 比較可能性メタデータの付与
7. 公開ページ、静的出力、Production Smoke

全7地域バッチ・38県で品質ゲート2まで完了しました。次工程は、Source Indexed県の指標本文と値を全件抽出し、Evidence付きReviewedデータへ昇格することです。Source IndexedはReviewedや独自の政策達成判定を意味しません。

## Comparability metadata

全国比較には、指標定義、単位、観測時点、母集団、単年・累計、実数・率・指数、目標型、速報・確報、再掲・内数・重複の一致または調整可能性が必要です。一つでも重要な不一致が残る指標は`not_comparable`としてランキングへ含めません。

## Missing-value rules

数値としての0、未設定、未公表、資料上のダッシュ、対象外、算定不能、未索引、定義変更により接続不能を別状態として保持します。

## Update and history rules

- 計画改定時は旧レコードを上書きしない。
- 計画期間、改定日、後継計画、失効状態を保持する。
- 同名指標でも定義、単位、母集団、目標年度が変われば別バージョンにする。
- 年度実績は、その時点で有効だった目標バージョンへ接続する。
- 最新計画の発見だけで旧計画の年度実績を新目標へ付け替えない。

## Completion gate

Phase 9は次をすべて満たした場合だけ`complete`になります。

- 47都道府県すべてで主要計画と主要数値目標がIndexed以上
- 公開する全数値のEvidence coverage 100%
- 再掲、欠損、単位、期間、母集団、目標型の品質テスト成功
- 更新周期、改定、後継・旧計画の履歴を追跡
- 比較不能な指標をランキングへ含めないことを機械検証
- 全国公開ページの静的出力とProduction Smoke成功
