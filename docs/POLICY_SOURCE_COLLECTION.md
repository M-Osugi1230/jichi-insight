# Policy source collection

## Purpose

Jichi Insightは、評価に必要なすべての資料が揃うまで公開を待つのではなく、自治体がすでに公開している政策資料を収集し、どこまで接続できるかを段階的に可視化します。

資料を登録したことと、内容を抽出したこと、事業へ接続したこと、成果を評価したことは分けます。

## Accountability chain

```text
Strategic plan
  自治体が目指す姿、目標、施策体系
      ↓
Implementation plan
  具体的な事業、期間、優先順位、計画事業費
      ↓
Annual priority program
  当該年度の重点施策と事業
      ↓
Progress / project review
  KPI、実績、課題、見直し
      ↓
Budget / settlement / contract
  予算、執行、決算、契約、支払先
```

## Collection states

| State | Meaning |
|---|---|
| `indexed` | 公式資料の所在を確認 |
| `ready_for_extraction` | 抽出対象と資料範囲を定義済み |
| `extracted` | 候補値・記述を抽出済み、レビュー前 |
| `reviewed` | 一次資料と照合済み |

初期10資料はすべて`ready_for_extraction`です。これは資料の公式性と所在を確認した状態であり、記載内容を政策実績として確認した状態ではありません。

## Publication boundary

- 資料件数を政策成果件数として表示しない
- 計画上の目標を実績として表示しない
- 予算額を成果として評価しない
- KPIの定義・基準値・目標値・実績値を分ける
- 年度、会計区分、資料版を保持する
- 原文位置を確認できない抽出値はReviewedへ昇格しない
- 事業と公約の対応は、名称の類似だけで推測しない

## Next milestone

最初の実データVertical Sliceは、1つの政策領域について次を接続することです。

```text
計画上の目標
  → 実施計画の事業
  → 年度予算
  → 実績・KPI
  → 事業点検または進捗報告
  → 根拠資料
```

この接続が再利用可能であることを確認してから、対象政策と自治体を拡大します。
