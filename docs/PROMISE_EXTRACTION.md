# Promise extraction and assessment rules

## Purpose

公約資料から記録を作るとき、文章を細かく分けるほど件数は増えます。しかし、件数を増やすための分割は、候補者が約束していない内容を作り出す危険があります。

Jichi Insightは、公約数を増やすことより、原文の境界と評価可能性を守ることを優先します。

## Source hierarchy

1. 選挙管理委員会が公表する選挙公報
2. 候補者本人または公式選挙事務所が公表したマニフェスト
3. 当選後に自治体が公表した政策方針・所信表明
4. 報道記事や第三者の要約

3と4は、選挙時の公約原文の代替として扱いません。補足資料または実行説明として別に保存します。

## Statement types

### specific_commitment

対象、行為、制度変更、数量、期限などの少なくとも一つが明確な約束。

### directional_commitment

政策の方向は示すが、対象・手段・期限の一部が曖昧な約束。

### vision

望ましい都市像や将来像。実施行為が特定できないため、進捗評価はしません。

### supporting_context

経歴、問題認識、政治姿勢、決意、実績説明。公約として採点しません。

## Segmentation rules

- 見出し、箇条書き、独立した文など、資料上の境界を優先する
- 句読点だけを理由に一つの文章を複数公約へ分割しない
- 一つの文に複数の行為があっても、資料上で一体なら最初は一記録として保持する
- 要約から原文にない数値、期限、対象、財源を追加しない
- 判読できない文字を推測で補完しない
- 長い文章は全文転載せず、短い原文部分と資料位置を保存する
- 分割判断に自信がない場合は、資料だけ登録し、公約レコードを作らない

## Field rules

### original_text

資料にある短い原文。要約文や解釈を書かない。

### source_location

ページ、候補者欄、見出し、段落など、第三者が同じ箇所を確認できる位置。

### baseline / target / deadline

資料に明示された場合だけ記録する。任期満了日を暗黙の期限として補わない。

### funding_statement

財源、予算規模、費用負担が明示された場合だけ記録する。「既存予算で対応」等も原文にない場合は補わない。

### controllability

自治体単独で実行できるか、国・県・議会・民間との共同が必要かを示す。不明な場合は `not_assessable` とする。

### assessability

- `assessable`: 進捗判定に必要な対象・行為・根拠がある
- `partially_assessable`: 一部だけ検証できる
- `not_assessable`: 理念、背景説明、または必要条件が不足

### progress_state

実行資料、予算、事業、KPI、公式説明へ接続するまで `not_assessable` とする。選挙公報に掲載された事実だけで `not_started` と判定しない。

## Mandatory safety rules

- 理念と背景説明は `not_assessable` 以外にしない
- 政策思想の賛否を採点しない
- 公約変更を自動的に失敗と判定しない
- 外部要因と自治体権限を確認せず成果を首長へ帰属しない
- 原文、要約、解釈、進捗評価を別項目にする
- 不明・未登録・未評価をゼロや未達へ変換しない

## Current Kitakyushu status

2023年北九州市長選挙の公式選挙公報は資料単位で登録済みです。候補者欄は政策方向と経歴・決意を含む文章であり、独立した公約境界の人手レビューが必要です。

現時点では個別公約レコードを作成せず、進捗評価を0件のまま維持します。
