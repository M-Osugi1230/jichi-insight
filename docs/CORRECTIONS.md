# Corrections and right of reply

## Purpose

Jichi Insightは、誤りをゼロと断言するのではなく、誤りを見つけ、検証し、訂正し、その履歴を残せる仕組みを公開します。

訂正は信頼を損なう例外処理ではなく、信頼を維持する通常運用です。

## What can be submitted

### Factual correction

- 金額、年度、単位、名称、日付の誤り
- 予算、執行、決算の区分誤り
- 一般会計、普通会計、特別会計等の区分誤り
- 公約原文、議案、採決、発言の誤記
- 出典URL、ページ番号、資料名の誤り

### Missing or stronger evidence

- より直接的な一次資料
- 差し替え後の公式資料
- 公表済みだが未収録の資料
- 現在の評価を変更し得る公式説明

### Right of reply

自治体、首長、議会、議員、政党、関係機関は、掲載内容について公式見解または補足資料を提出できます。

事実訂正と意見・反論は区別して扱います。

## Required information

- 対象ページまたはレコード
- 現在の記載
- 修正を求める内容
- 公式資料URL
- 資料名、年度、該当ページ
- 申請者の立場（任意）

根拠資料がない意見も受け付けますが、事実訂正としては処理しません。

## Severity

| Level | Example | Initial target |
|---|---|---|
| P0 | 投票判断や公費評価を重大に誤らせる可能性 | 24時間以内に確認開始 |
| P1 | 重要な金額、年度、対応付けの誤り | 3営業日以内に確認開始 |
| P2 | 軽微な表現、表記、メタデータ | 10営業日以内に確認開始 |

これは対応目標であり、複雑な資料確認や関係者照会により判断まで時間を要する場合があります。

## Review process

```text
受付
  → 申請内容と個人情報の確認
  → 一次資料の確認
  → 既存Evidence Packetとの比較
  → 事実訂正 / 評価変更 / 反論掲載 / 対応なしを判定
  → PRで修正と検証
  → 変更履歴を公開
```

## Change history

重大な訂正では、次を残します。

- 修正前
- 修正後
- 修正理由
- 根拠資料
- 修正日時
- 影響した評価・画面

誤りを静かに差し替えません。

## Privacy and security

公開Issueに、自宅住所、電話番号、非公開メール、家族情報、認証情報、未公開資料を投稿しないでください。

脆弱性や機密情報は、GitHubのPrivate vulnerability reportingを使用します。

## Independence

訂正申請者の立場、政党、自治体、支持・不支持によって基準を変更しません。

訂正を受け入れない場合も、判断理由と確認した資料を記録します。
