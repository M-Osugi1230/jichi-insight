# Source requests

## Purpose

公開サイトや通常のウェブ検索で一次資料を特定できない場合に、行政機関へ確認するための照会案を管理します。

照会案の作成と送信は別の状態です。文案を作成しただけで、行政機関へ問い合わせたとは表示しません。

## Status

- `draft`: 文案作成中。未送信
- `ready_for_review`: 宛先・文面の人手確認待ち。未送信
- `sent`: 送信済み。回答待ち
- `answered`: 回答を受領
- `no_record`: 保有していない、保存期間満了などの回答を受領
- `closed`: 対応終了

## Safety rules

- `draft`と`ready_for_review`は`sent_at=null`を必須とする
- 未送信状態では回答日時と回答内容を保存しない
- `sent`以降は送信日時を必須とする
- `answered`と`no_record`は回答日時と回答概要を必須とする
- 公開ページに送信者個人の氏名、住所、電話番号、メールアドレスを載せない
- 連絡先は行政機関が公式ページで公開した組織連絡先だけを使用する
- 送信は、利用者の明示的な指示またはプロジェクトで定めた承認手続の後に行う

## Request scope

最初の照会案では、次だけを確認します。

1. 選挙公報を保存しているか
2. 候補者提出原稿を保存しているか
3. 公約原文を確認できる公式資料があるか
4. ウェブ閲覧、窓口閲覧、写し交付の方法
5. 情報公開請求が必要か

行政機関に政策評価や候補者の意図の説明を求めません。資料の保有・公開・閲覧方法に限定します。

## Publication

公開サイトでは以下を区別します。

- 照会案作成済み
- 送信前レビュー中
- 送信済み
- 回答受領済み

未送信の照会案を「問い合わせ済み」「回答待ち」と表現しません。
