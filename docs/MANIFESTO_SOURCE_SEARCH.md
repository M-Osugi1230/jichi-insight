# Manifesto source search

## Purpose

公約原文資料を確認できない場合に、単に「未登録」とするのではなく、確認した範囲、結果、次の調査方法を記録します。

## Core distinction

`no_stable_primary_source_found` は、資料が存在しないという意味ではありません。

- 確認した範囲では登録可能な一次資料を特定できなかった
- 非公開、削除済み、紙媒体のみ、別URL、ウェブアーカイブ外などの可能性が残る
- 後から資料が見つかった場合は探索記録を更新し、過去の判断を消さない

すべての探索記録は `nonexistence_claim=false` を必須とします。

## Result states

- `source_registered`: 公約原文資料を登録済み
- `official_result_only`: 公式選挙結果までは確認したが、公約資料は未登録
- `no_stable_primary_source_found`: 複数の探索範囲で安定した一次資料を特定できない
- `follow_up_required`: 照会、アーカイブ確認、候補者資料確認などの追加作業が必要

## Search scopes

- 個別選挙結果ページ
- 自治体公式サイト内検索
- 公式文書アーカイブ
- 候補者本人の公表資料
- 一般ウェブ検索

検索語の一致だけで資料を採用しません。発行主体、対象選挙、候補者、公開状態、安定URLを確認します。

## Publication rules

公開画面では次を分けます。

1. 選挙結果を確認済み
2. 公約資料を探索済み
3. 公約原文資料を登録済み
4. 公約単位への分割レビュー済み
5. 個別公約レコード作成済み
6. 進捗評価済み

上位段階が未完了でも、下位段階の件数を完成度や評価点として扱いません。
