# Data quality and publication readiness

## Purpose

Jichi Insightは、収録件数だけで進捗を表しません。資料を見つけた状態、値を抽出した状態、人が一次資料と照合した状態、公開基準を通過した状態を分けます。

## Quality ladder

| Level | Meaning | Public use |
|---|---|---|
| Coverage | 対象・資料の存在を把握 | 収録候補の表示のみ |
| Indexed | 公式URL、資料名、年度、位置を特定 | 公式資料カタログ |
| Extracted | 候補値を抽出 | 原則非公開 |
| Reviewed | 一次資料と人が照合 | 事実表示に利用可能 |
| Verified | 独立した二重確認を完了 | 重要評価の主要根拠 |
| Published | 公開基準と画面検証を通過 | 本番サイト |

`Reviewed`や`Verified`は情報が正しいと永久に保証するものではありません。訂正・差し替え・新資料に応じて再確認します。

## Why evaluations remain zero

財政値が確認できても、自治体や首長を評価するには人口・行政需要を踏まえた比較、事業別予算・契約・執行・決算、KPI、公約、議会資料、外部要因の検討が不足しています。

不足を点数で埋めず、「評価不能」または「未評価」と表示します。比較ページも規模の違いを優劣へ変換しません。

## Quality metrics

- 公式資料リンク率
- Reviewed / Verified比率
- Evidence Packet coverage
- 最終確認日とデータ鮮度
- リンク切れ
- スキーマエラー
- 参照切れ
- 訂正件数と解決時間
- 自治体・政策分野別の欠損
- 比較指標の年度・会計・段階整合率

## Finance phase completion

初期3自治体すべてで、少なくとも当初予算または決算のReviewed財政値を公開し、全財政値にEvidence Packetを接続しました。

次工程では、金額の総額から個別の政策へ進みます。

1. 重点事業の目的と対象者
2. 当初予算・補正・執行・決算
3. 契約方式・契約先・金額
4. KPIの基準値・目標・実績
5. 監査・議会質疑・翌年度対応
6. 首長公約との対応

## Expansion gate

自治体数を大きく増やす前に、パイロット自治体で「財政 → 重点事業 → 契約 → KPI → 公約 → 議会」を一つの流れとして接続し、同じ品質基準を再利用できることを確認します。
